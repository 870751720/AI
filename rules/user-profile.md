# User Profile — 行事风格与决策偏好

> 本文件记录项目维护者的核心工作风格，供所有 AI Agent 读取并遵循。

## 1. 启动风格：规划驱动

- **先想清楚再动手**。面对模糊需求时，不盲目开干，优先梳理边界、异常和架构后再编码。
- 需要我在动手前先出方案或架构给你确认。

## 2. 技术债：零容忍

- **几乎不接受技术债**，宁愿慢也要一次性做对。
- 核心模块和边缘模块都应保持干净，不因为"能跑就行"而妥协。

## 3. 决策模式：信息透明，你掌握决策权

- 当我遇到不确定（如两个方案各有利弊）时，**必须列清利弊，等你拍板**。
- 不要默认替我做重大决策，但可以给出推荐并说明理由。

## 4. 沟通密度：关键节点同步

- **关键决策同步，执行细节自由发挥**。
- 你信任我的执行，但大方向、架构变动、风险点必须汇报。
- 只汇报里程碑和 blocker，不需要每一步都确认。

## 5. 风险应对：遇阻即停，重新评估

- 项目出现计划外问题（如重构比预期麻烦）时，**立即暂停，重新评估**整体计划。
- 不硬扛、不擅自延长时间盒，先停下来讨论再继续。

## 6. 质量要求：严格统一，文档留存

- 代码风格严格统一，核心逻辑必须有注释，命名要清晰。
- **重要变更需留存设计文档**或架构说明，方便后续维护。

## 7. 长任务：信任驱动，一次性汇报

- 对于超过 20 分钟的长任务，**完成后一次性汇报结果**，中间不打扰。
- 但如果中途发现明显偏离预期，仍需立即暂停确认。

---

**一句话画像**：规划驱动、质量洁癖、风险敏感、掌控关键决策但信任执行细节。
