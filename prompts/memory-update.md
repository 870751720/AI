# Memory Update Prompt — 记忆更新提示词

> 本模板供 AI Agent 在长任务结束或被用户纠正后使用，用于更新记忆文件。

---

## 何时触发

1. **长任务结束**（耗时 > 20 分钟）
2. **用户明确纠正**（"不对"、"错了"、"应该…"）
3. **架构/重要决策变更**
4. **踩坑或发现意外问题**

---

## 更新流程

### Step 1：判断更新目标

| 场景 | 目标文件 | 操作 |
|------|---------|------|
| 被纠正 | `memory/learnings.md` | 追加：错误表现 → 正确做法 → 场景 |
| 架构/配置变更 | `memory/project-context.md` | 追加到「重要变更记录」 |
| 长任务完成 | `memory/sessions/YYYY-MM-DD-任务名.md` | 写会话摘要 |
| learnings.md > 200 行 | `memory/sessions/归档-日期.md` | 旧内容迁移 + 生成摘要 |

### Step 2：会话摘要模板（sessions/）

```markdown
# 会话摘要 — [任务简述]

**日期**：YYYY-MM-DD  
**耗时**：约 X 分钟  
**关联文件**：`fileA.ts`, `fileB.ts`

## 目标

<!-- 本次任务要解决什么 -->

## 关键决策

<!-- 做了哪些架构或实现决策 -->

## 修改内容

<!-- 改了哪些文件，核心逻辑是什么 -->

## 风险点 / 待确认

<!-- 有没有遗留问题或需要用户确认的 -->

## 下次继续时的切入点

<!-- 如果任务没做完，从哪里接着干 -->
```

### Step 3：learnings 追加格式

```markdown
### [简短标题]（YYYY-MM-DD）

- **错误表现**：...
- **正确做法**：...
- **触发场景**：...
```

---

## 原则

- **简洁**：只记关键，不记流水账
- **可搜索**：标题要包含关键词，方便日后扫描
- **可执行**：正确做法要具体，不能含糊
- **透明**：用户随时可读、可改、可删
